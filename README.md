# CCSF — Reproducibility package

Code and synthetic data accompanying the manuscript:

**"Compliance-Constrained Semantic Fingerprinting for Detecting Coordinated
Synthetic Advocacy in Alternative Nicotine Delivery System Discourse:
A Synthetic Proof-of-Concept Study."**

All data in this package are **synthetically generated** for methodological
evaluation. There are no human participants, no real accounts, and no scraped
platform data. Everything is deterministic given the fixed random seed
(`SEED = 20260604`).

> **Scope.** This is a controlled simulation establishing internal construct
> validity under known ground truth. The synthetic campaign was generated by
> rules designed to instantiate the very regularities the detector measures, so
> the reported AUCs are a construct-validity check, **not** an estimate of
> real-world operational accuracy. Validation on independent generators and on
> labelled or expert-adjudicated real platform data is the essential next step.

## Contents

| File | Description |
|------|-------------|
| `corpus_gen.py` | Generates the synthetic corpus: 524 posts from 78 simulated accounts across 8 ANDS policy topics, in four groups (organic, coordinated synthetic, professional, human-edited synthetic = `mixed`). Defines the compliance lexicon and all text templates. |
| `analysis.py` | Feature extraction and statistics. Computes per-post language-model perplexity (GPT-2), sentence embeddings (all-MiniLM-L6-v2), stance, and account-level aggregates; runs Mann-Whitney U + Cliff's delta, single-feature and composite ROC/AUC (with bootstrap 95% CIs), 5-fold logistic-regression CV, leave-one-out ablation, KMeans/DBSCAN clustering, and the non-duplication (Jaccard vs cosine) check. Writes results.json, arrays.npz, synthetic_corpus.csv and account_features.csv. |
| `figures.py` | Renders the five manuscript figures (300 dpi PNG) from results.json / arrays.npz. All plotting randomness (jitter) is seeded. |
| `verify_outputs.py` | Checks data-generation invariants and that the headline numbers in results.json match those reported in the manuscript (prints PASS/FAIL; non-zero exit on failure). |
| `results.json` | All numeric results reported in the manuscript. |
| `arrays.npz` | Intermediate arrays: post-level embeddings (524x384), perplexity, compliance, stance; account-level features for all 78 accounts; composite anomaly scores. |
| `synthetic_corpus.csv` | Full corpus: id, account, group, topic, text (regenerated by analysis.py). |
| `account_features.csv` | Account-level feature table (regenerated by analysis.py). |
| `fig1_pipeline.png` ... `fig5_nonduplication.png` | The five figures. |
| `requirements.txt` / `requirements-lock.txt` | Minimal and exact-version dependencies. |
| `LICENSE`, `CITATION.cff` | MIT (code) / CC BY 4.0 (synthetic data); citation metadata. |

`mixed` in the code/data corresponds to **"human-edited synthetic"** in the manuscript.

## Requirements & setup

Python 3.10+ (tested on 3.12.3). Install dependencies:

```bash
pip install -r requirements.txt          # minimal
# or, to reproduce the published numbers exactly:
pip install -r requirements-lock.txt
```

The first run downloads two pretrained models from the Hugging Face Hub:
`gpt2` (~0.5 GB) and `all-MiniLM-L6-v2` (~0.1 GB). **Internet access is needed
once** for this download; afterwards the models are cached
(`~/.cache/huggingface`) and runs are offline. Set `HF_HUB_OFFLINE=1` to force
use of the cache.

**Runtime.** On a modern CPU (no GPU required) the full pipeline runs in roughly
3-6 minutes, dominated by GPT-2 perplexity scoring of 524 posts.

## Reproduce everything

```bash
python analysis.py     # -> results.json, arrays.npz, synthetic_corpus.csv, account_features.csv
python figures.py      # -> fig1..fig5 .png
python verify_outputs.py   # -> PASS/FAIL on data invariants and headline numbers
```

All scripts read and write relative to their own directory, so they can be run
from any working directory. Because every source of randomness is seeded
(including figure jitter and the AUC bootstrap), re-running reproduces the
corpus, statistics and figures. Small floating-point differences in perplexity
are possible across different torch/transformers builds, but group-level effect
sizes and AUCs are stable; use requirements-lock.txt for exact reproduction.

corpus_gen.py can also be called directly:

```python
import corpus_gen
posts = corpus_gen.build_corpus()   # list of dicts: id, account, group, topic, text
```

## Notes on specific design choices

- **Compliance density** counts, per word, the occurrences of a fixed list of
  industry/commercial-framing phrases, matched independently on word boundaries
  (case-insensitive). A longer listed phrase that contains a shorter listed
  phrase counts both, by design, because each is treated as an independent
  framing cue. Implemented in analysis.py:compliance_density.
- **Composite anomaly score** is the *mean* of five oriented, standardised
  account-level features: lower perplexity, lower burstiness, higher compliance
  density, lower stance variance, higher embedding convergence. Within-account
  perplexity *variance* is reported descriptively but excluded from the composite
  to avoid double-weighting the perplexity family.
- **Convergence** is the mean pairwise cosine similarity among an account's own
  post embeddings (intra-account), not similarity to a global campaign centroid.
- **t-SNE (Figure 3)** is used for visualisation only; all quantitative analyses
  are account-level.
